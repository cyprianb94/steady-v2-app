Training pace picks the zone. Effort is what it feels like.

Three sections — TYPE, INTENSITY, then VOLUME or THE SET — replace the flat row stack. The training-pace row offers only zones that make sense for the chosen session type (Easy never offers threshold; Interval never offers recovery). The effort cue — conversational, hard repeatable — is derived from the zone and shown as a caption next to the pace value, never as a chip the runner has to pick.

The fix · pace zones vs effort cues

"Effort: interval" was nonsense. Now the row is Training pace.

Pace zones come from the runner's goal time and are quantitative: recovery / easy / steady / marathon / threshold / interval. Effort cues are qualitative — conversational, hard repeatable, sharp. The previous mockup conflated them. Now the chip row contains only zones, and the cue is shown next to the pace value as a small effort-plum caption — feels conversational, feels hard repeatable. Derived, not picked.

Zones filtered by session type

Easy never offers threshold. Interval never offers recovery.

Easy: recovery · easy · steady. Interval: marathon · threshold · interval. Tempo would be: steady · marathon · threshold. Long would be: recovery · easy · steady · marathon. The active zone is the default for that session type, so changing type swaps in the right zones automatically and pre-selects the sensible one. The runner never sees a list of irrelevant choices.

Section name shifts with type

VOLUME for distance sessions. THE SET for intervals.

Easy / Long / Tempo use VOLUME — distance is the size of the run. Interval uses THE SET — reps × length × recovery. "The set" is what a coach actually says when they write 6×800m, 90s rest. Naming the section that way matches the runner's mental model rather than forcing one umbrella label across very different shapes of session.

Bookends & unit toggles

KM/MIN appears only where both units are genuinely common.

Only Interval and Tempo expose bookends — Easy and Long don't, per the data model. Each row defaults to the relevant unit (km, not min) and hides the toggle behind Custom…. The KM/MIN pill only shows where both units make sense: Each rep (400m vs 90s) and Recovery between reps (60s vs 100m). Warm-up and cool-down stay km-only.

Colour discipline

Each colour appears once, in one place.

Forest only on the Easy tag. Clay only on the Interval tag. Cobalt only on the distance value. Teal only on the pace value. Effort plum only on the cue caption. Selected controls go to ink, not to a metric colour, so the chip strip never competes with the value above it. Every colour answers exactly one question.

Save model

Cancel/Save in the nav bar; propagation is a quiet footer.

The full-width clay Update session button is gone in both variants. Cancel/Save are paired in the nav bar following iOS convention. The propagate-modal (this Tuesday only / all remaining weeks / build phase only) becomes a one-line muted footer warning so the runner isn't surprised by a modal after tapping save.
