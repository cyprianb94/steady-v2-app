08 · Niggle picker

  • Four-step wizard on one screen. All four groups visible at once, no multi-page flow — each is a constrained choice set so it never overflows.
  • Body part grid — 12 fixed options from a closed enum. Selected pill uses the clay accent (--clay) to hint "this is the injury/caution signal" without alarming.
  • Severity ramp: Niggle → Mild → Moderate → Stop. Colours escalate across the row — first two on --amber (caution), third on --clay-bg (concern), fourth filled --clay (stop). This is the only spot in the app where a small red-ish accent is allowed.
  • Sub-labels under each severity chip describe what it actually means — "noticed", "ran through", "eased off", "had to stop". Makes the scale self-explanatory without a tooltip.
  • "When" row — before / during / after maps to the exact shape of the data model: Niggle.when.
  • Fixed bottom CTA — ink pill because the accent is already carried by the severity tiles above. Avoids colour collision.
  • Data shape on save: { bodyPart: 'hamstring', side: 'left', severity: 'mild', when: 'during' } — appended to subjectiveInput.niggles.
  • Why this matters: directly serves the recovery/injury USP. Structured niggle history becomes the input for future Steady AI warnings ("Left hamstring has flagged 3× this week — consider an easier block").
