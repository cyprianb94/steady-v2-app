05 · Match picker

  • Reached from: tapping the "matched to today" chip on screen 03 OR the "Match to a session ›" CTA on screen 04.
  • Run summary card at top — keeps the user anchored on which run they're matching while scrolling through candidates.
  • Grouped options: "This week" lists planned sessions (prioritised by proximity + match score from matchActivity()). The top candidate is pre-selected and tagged AUTO.
  • Tag colours:
    • AUTO — forest, the system's suggestion
    • MISSED — clay, sessions without an activity yet
    • NEUTRAL — slate, "bonus run"
  • Selected card gets a 1.5px forest border and forest-bg fill; radio dot fills in forest.
  • Confirm in the nav bar (not a full-width CTA) because the user has already committed psychologically by tapping a radio.
  • Backend: tapping Confirm calls activity.setMatchedSession(activityId, sessionId | null). Server re-runs plan state.
