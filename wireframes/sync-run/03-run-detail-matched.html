03 · Run Detail — matched (core)

  • This is the core screen. Reached after a successful sync when exactly one Strava activity matches today's planned session.
  • Narrative order (top → bottom): identity → headline metrics → gear → splits → plan vs actual → feel → niggles → notes → save. This is the order a runner actually wants to review.
  • Matched chip at the top uses --forest (EASY accent) and is tappable — opens screen 05 (re-assign match) if auto-match is wrong.
  • Metric cells 2×2 grid. Values in Space Mono 24/700; units in sans at 13px muted. Labels uppercase sans 10px with 1.3px tracking.
  • Shoe row — auto-populated from Strava gear_id on sync. Shows lifetime km pulled from the Shoe entity. "Change ›" opens screen 07.
  • Splits list — uses existing Activity.splits. Bar width = relative pace rank (fastest = 100%). Forest accent on fills. No new data needed.
  • Planned vs actual — rendered from summariseVsPlan(session, activity) in packages/types/src/plan-vs-actual.ts. Deterministic, no LLM. Verdict lines are Verdict[]; 3-line max for visual calm.
  • Subjective — extracted from TodayHeroCard.tsx:254. Same 3 options, bigger layout. Pills are --ink when selected, translucent white otherwise.
  • Niggles — first-class. Tapping ＋ opens screen 08. Each niggle row is clay-bg to read as "caution," not alarm. X removes.
  • Notes — dashed-border writing zone, hints at "a line or two." Persists as Activity.notes.
  • Save CTA — session-coloured full-width pill. Same colour as the Home CTA on screen 01 to close the loop visually.
  • Re-entry: after saving, home shows "Review run ›" which re-opens this screen in editable mode.
