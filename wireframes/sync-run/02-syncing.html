02 · Syncing overlay

  • Trigger: user tapped FinishedRunCta. Overlay fades in above Home, Home blurs and desaturates behind.
  • Spinner: 56px, 3px ring, animated with top-colour --forest (accent of the matched/current session).
  • Copy: terse and honest — tells the user where data is coming from and the rough duration.
  • Three-step tracker maps to the real sync stages:
    1. CONNECTED — token exchange ok (usually already true).
    2. FETCHING — strava.sync mutation in flight (strava-client.getActivities).
    3. MATCHING — matchActivity() running against planned sessions.
  • Error states (not drawn): Strava 401 → "Reconnect Strava"; 0 new runs → empty-state card ("No new run found"); matcher finds >1 activity → screen 06.
  • Duration: min 800ms visible even if sync is instant, to avoid a jank.
