07 · Shoe picker

  • Auto-populated list from shoe.list — shoes upserted during Strava sync via /api/v3/gear/{id}.
  • Lifetime km bar is a free side-effect of tracking gear. Colour shifts with wear:
    • --forest under 60% of retirement distance
    • --amber 60–85% (warning band)
    • --clay 85%+ (replace soon)
    This becomes a natural hook for the recovery/injury USP — shoe wear + niggles correlate.
  • "FROM STRAVA" tag on clay-bg makes the source clear so users know to update it in Strava if wrong.
  • Retired group — grey-out. Tapping a retired shoe is allowed (for back-dating), just visually de-emphasised.
  • ＋ Not tracked / add new shoe — dashed-border button. v1 just sets shoeId = null on the activity. Adding a new shoe manually is a v2 enhancement.
  • Backend: Done → activity.setShoe(activityId, shoeId | null). Future: we can also decrement/increment lifetime km server-side as the single source of truth.
